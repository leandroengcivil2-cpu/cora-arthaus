TABELAS SUPABASE:
- empreiteiros: id TEXT PK, nome TEXT, especialidade TEXT, cnpj TEXT
- colaboradores: id TEXT PK, nome TEXT, empreiteiro_id TEXT, funcao TEXT, cpf TEXT
- chamadas: id TEXT PK, data DATE, colaborador_id TEXT, status TEXT (P/F/M), motivo TEXT
- previsto: id TEXT PK, mes TEXT (YYYY-MM), empreiteiro_id TEXT, quantidade INTEGER
- funcoes: id TEXT PK, nome TEXT
- diarias: id TEXT PK, data DATE, colaborador_id TEXT, descricao TEXT
- Constraints: UNIQUE(data, colaborador_id) em chamadas e diarias

FUNCIONALIDADES DO APP:
1. Home: Dashboard com efetivo hoje, colaboradores, empreiteiros, imagem do prédio Corá à direita
2. Chamada: Marcação diária P/F/M com campo de motivo quando F ou M
3. Painel BI: Gráfico previsto x realizado por empreiteiro (barras verticais lado a lado), tabela diária, gráficos de efetivo por função (barras verticais com nome da função dentro), exportação PDF mensal e diária
4. Diárias: Seletor para adicionar colaborador no dia, campo de descrição do serviço, exportação PDF medição mensal
5. Cadastros: Empreiteiros (CNPJ obrigatório), Colaboradores (CPF obrigatório), Funções, Previsto mensal

REGRAS DE NEGÓCIO OBRIGATÓRIAS:
- Todos os campos de texto forçam CAIXA ALTA
- CNPJ obrigatório e único (14 dígitos com máscara)
- CPF obrigatório e único (11 dígitos com máscara)
- Selects NÃO devem aplicar toUpperCase (corrompe IDs)
- Queries de data devem usar último dia real do mês (NUNCA dia 31 fixo)
- Todo POST ao Supabase deve gerar campo id
- Filtro de empreiteiro no BI deve ser respeitado nos PDFs
- Nenhum uso do termo HD ou homem-dia em nenhum lugar
- PDFs com assinatura engenheiro + empreiteiro
- Campo motivo aparece quando status é F ou M na chamada
- Interface em português do Brasil, sem emojis, sem inglês
